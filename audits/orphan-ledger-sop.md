# Orphan Ledger SOP

Every cowork session running a BUILD prompt in the Patient Identity Network track must treat `audits/orphan-ledger.md` as a hard contract: read it at the start of the prompt to understand which artifacts are still incomplete and which prompt is on the hook for closing them; update it at the end of the prompt's deliverable by adding any new partial-slice artifacts to "Open Items" with a unique ID, type, creating prompt, closing prompt, owner, and notes, and by moving any items this prompt fully closed (DB schema + RLS + data layer + API + UI + i18n + tests all in place) from "Open Items" to "Closed Items" with the verifying test recorded. Before declaring the prompt done, perform a validation pass — re-read the ledger and confirm every Open row added by this prompt has a sensible Closing Prompt assignment (not blank, not the same prompt that opened it unless the slice was fully closed inline, and not later than Prompt 11), and confirm no Open rows were silently deleted. The ledger is the single source of truth that the final E2E test prompt will check; if it isn't kept current, the build breaks at the end.
